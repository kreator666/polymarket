## 核心工具：`find_deep_markets.py`

### 功能

自动扫描 Polymarket 全站活跃市场，筛选出**订单簿深度充足**（买卖双侧均可承受一定规模交易而不产生过大滑点）的标的，用于：

- **量化策略标的池构建** — 只交易有足够对手盘的合约
- **做市标的选择** — 避免进入流动性枯竭的陷阱市场
- **人工交易辅助** — 快速定位当前深度最好的市场

### 筛选原理

1. **Gamma API** 按 `liquidity` 排序获取头部活跃事件（默认 Top 50）
2. 提取所有启用了订单簿（`enableOrderBook`）的市场，解析 `clobTokenIds`
3. **CLOB API** 并发查询每个市场的实时订单簿（10 线程）
4. 计算**有效深度**：
   - **买入深度**：在最优 Ask 价基础上，累加容忍 `MAX_SLIPPAGE` 滑点范围内的挂单
   - **卖出深度**：在最优 Bid 价基础上，累加容忍 `MAX_SLIPPAGE` 滑点范围内的挂单
5. 过滤掉价差过大（`spread > MIN_SPREAD_FILTER`）的市场——知识库经验表明此类市场流动性极差，显示价格实为最近成交价而非有效盘口
6. 输出**买入深度与卖出深度均 ≥ `DEPTH_THRESHOLD`** 的 Top 10 标的

### 快速开始

```bash
# 依赖：仅需 requests（Python 3.8+）
pip install requests

# 运行（默认筛选 Sports 类别，深度阈值 1000 USDC）
python find_deep_markets.py
```

### 配置参数

在 `find_deep_markets.py` 顶部修改：

| 参数 | 默认值 | 说明 |
|---|---|---|
| `DEPTH_THRESHOLD` | `1000.0` | 有效深度阈值（USDC），低于此值的市场将被过滤 |
| `MAX_SLIPPAGE` | `0.02` | 可接受滑点（美元），计算深度时累单的价格容忍区间 |
| `TOP_EVENTS` | `50` | 初始扫描的流动性头部事件数量，可适当增大以覆盖更多市场 |
| `MAX_WORKERS` | `10` | 并发查询 CLOB 订单簿的线程数 |
| `MIN_SPREAD_FILTER` | `0.10` | 最大允许价差，超过则视为流动性不足并跳过 |
| `CATEGORY_FILTER` | `"Sports"` | 类别过滤，支持 `"Sports"`、`"Politics"`、`"Crypto"`、`"Culture"` 及子类别（如 `"NBA"`），设为 `None` 表示全市场扫描 |

## 知识库：`polymarket_cn_kb/`

离线收录了 [Polymarket 官方中文文档](https://docs.polymarket.com/cn) 的完整内容（截至 2026-04），共 **50 个页面**，便于本地检索、快速查阅与 LLM 问答。

### 内容覆盖

| 模块 | 内容 |
|---|---|
| **入门指南** | Polymarket 101、快速入门、开发者快速上手（TypeScript / Python） |
| **核心概念** | 市场与事件结构、订单簿与价格机制、持仓与代币（CTF）、订单生命周期、市场判定与结算 |

| **交易** | 交易概述、创建/取消/归因订单、Gasless 免 Gas 交易、撮合引擎原理、CTF 拆分/合并/赎回、跨链桥存取、费用结构 |
| **做市商** | 入门指南、库存管理、Maker 返利、流动性奖励计划 |
| **构建者计划** | API Keys 申请、 tiers 等级体系 |
| **API 参考** | 身份验证（Nonce + 签名）、速率限制、官方 Python / TypeScript SDK、地区限制说明 |
| **资源** | 合约地址、错误码对照表 |
| **高级主题** | Negative Risk 负风险市场 |

### 使用方式

- **全文搜索**：在 `polymarket_cn_kb/` 目录下对 Markdown 文件进行关键词检索
- **LLM 增强**：将相关文档作为上下文传入大模型，获取针对 Polymarket 开发的专业问答
- **离线阅读**：直接打开任意 `.md` 文件查阅，所有内容均为中文
